# 🌐 STALKnet Web Client

## 📋 Описание

**STALKnet Web Client** — веб-клиент для работы с децентрализованной чат-системой STALKnet.

Клиент обеспечивает:
- ✅ Real-time обмен сообщениями через WebSocket
- ✅ Авторизацию и регистрацию пользователей
- ✅ Сохранение сессии (7 дней)
- ✅ Мобильную адаптацию
- ✅ Тёмную тему оформления
- ✅ Историю команд
- ✅ Ответы на сообщения

## 🔐 Авторизация

### Требования к учётным данным

| Поле | Требование |
|------|------------|
| Имя пользователя | Минимум 2 символа |
| Пароль | Минимум 6 символов |

### Состояния авторизации

| Состояние | Код | Описание |
|-----------|-----|----------|
| Guest | 0 | Гость, не авторизован |
| EnteringName | 1 | Ввод имени |
| ConfirmCreate | 2 | Подтверждение создания профиля |
| EnteringPassword | 3 | Ввод пароля |
| Authorized | 4 | Авторизован |

---

### Процесс регистрации нового пользователя

```
1. /auth
   → "Введите имя сталкера:"

2. Введите имя (например: BG)
   → Проверка имени через API
   → Имя свободно → "Создать профиль? /y или /n"

3. /y
   → "Введите пароль для: BG"

4. Введите пароль (мин. 6 символов)
   → POST /api/auth/register
   → POST /api/auth/login
   → Сохранение сессии в localStorage
   → Подключение WebSocket
   → "Добро пожаловать, BG! Session ID: ...abc123"
```

---

### Процесс входа существующего пользователя

```
1. /auth
   → "Введите имя сталкера:"

2. Введите имя (например: BG)
   → Проверка имени через API
   → Имя найдено → "Введите пароль:"

3. Введите пароль
   → POST /api/auth/login
   → Получение JWT токенов
   → Сохранение сессии в localStorage
   → Подключение WebSocket
   → "Вход выполнен успешно!"
```

---

### Быстрый вход

```
/login <user> <pass>

Пример:
/login BG password123
```

---

### Выход из системы

```
/logout — выход из аккаунта
/quit — выход и прощальное сообщение
```

## 🔌 WebSocket

### Подключение

После авторизации клиент автоматически подключается к WebSocket:

```javascript
const ws = new WebSocket(
  `ws://${window.location.hostname}:8083/ws/chat?room_id=1&user_id=${userId}&username=${username}`
);
```

### Формат сообщений

**Отправка:**
```json
{
  "type": "message",
  "content": "Привет, сталкер!"
}
```

**Получение:**
```json
{
  "type": "message",
  "content": "Привет, сталкер!",
  "username": "BG",
  "user_id": 5,
  "timestamp": "2026-03-01T12:00:00Z"
}
```

### Типы сообщений

| Тип | Описание | Цвет |
|-----|----------|------|
| `message` | Сообщение пользователя | `#cdc0b0` |
| `system` | Системное сообщение | `#cccccc` |
| `task` | Уведомление о задаче | `#cf9c4e` |

## 💾 Сохранение сессии

### localStorage

Сессия сохраняется в `localStorage` на 7 дней:

```javascript
localStorage.setItem('stalknet_session', JSON.stringify({
  username: "BG",
  sessionId: "BG_abc123...",
  accessToken: "eyJhbGciOiJIUzI1NiIs...",
  refreshToken: "eyJhbGciOiJIUzI1NiIs...",
  authState: 4,
  userId: 5,
  savedAt: 1709294100000
}));
```

### Восстановление сессии

При загрузке страницы:
1. Проверка `localStorage` на наличие сессии
2. Проверка возраста сессии (< 7 дней)
3. Проверка валидности access токена
4. При необходимости — refresh токена
5. Автоматическое подключение WebSocket

## ⚠️ Известные ограничения

1. **Одна комната** — по умолчанию room_id=1
2. **Нет индикатора набора** — не показывается когда пользователь печатает
3. **Нет уведомлений** — браузерные push-уведомления не реализованы
4. **Нет загрузки файлов** — только текстовые сообщения
5. **История сообщений** — не сохраняется локально (только на сервере)
