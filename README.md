# STALKnet

## Сервисы

| Сервис       | Порт | Описание                              |
|--------------|------|---------------------------------------|
| Gateway      | 8080 | API Gateway, роутинг, аутентификация  |
| Auth         | 8081 | Регистрация, логин, JWT               |
| User         | 8082 | Профили пользователей, статусы        |
| Chat         | 8083 | WebSocket, комнаты, сообщения         |
| Task         | 8084 | Управление задачами                   |
| Notification | 8085 | Уведомления                           |

## Клиенты

### TUI Клиент (Консольный)

Запуск:
```bash
cd client
stalknet_client.exe
```

**Команды TUI:**
| Команда           | Описание             |
|-------------------|----------------------|
| `/help`           | Показать команды     |
| `/clear`          | Очистить экран       |
| `/nick <name>`    | Сменить имя          |
| `/connect`        | Статус подключения   |
| `/quit`           | Выйти                |
| `/mock <text>`    | Отправить сообщение  |
| `/mockmsg`        | Случайное сообщение  |
| `/mocktask`       | Уведомление о задаче |
| `/scroll up/down` | Прокрутка            |

**Управление:**
- `Enter` — Отправить
- `Ctrl+C` — Выход
- `Ctrl+L` — Очистить экран
- `PgUp/PgDn` — Прокрутка

### Веб-Клиент (Browser)

Запуск сервера:
```bash
cd client
stalknet_web.exe
# или
go run main_web.go
```

Открыть в браузере: **http://localhost:8080**

**Команды Веб:**
| Команда | Описание                    |
|---------|-----------------------------|
| `/help`        | Показать команды     |
| `/clear`       | Очистить экран       |
| `/nick <name>` | Сменить имя          |
| `/connect`     | Статус подключения   |
| `/quit`        | Выйти                |
| `/mock <text>` | Отправить сообщение  |
| `/mockmsg`     | Случайное сообщение  |
| `/mocktask`    | Уведомление о задаче |

**Управление:**
- `Enter` — Отправить
- `Ctrl+L` — Очистить экран
- Кнопка `SEND` — Отправить сообщение

**Настройка порта:**
```bash
set STALKNET_WEB_PORT=3000
stalknet_web.exe
```

## Быстрый старт

```bash
# Запуск всех сервисов
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановка
docker-compose down
```

## API Endpoints

### Auth Service
- `POST /api/auth/register` — Регистрация
- `POST /api/auth/login` — Вход

### Chat Service
- `WS /ws/chat` — WebSocket соединение для чата
- `GET /api/chat/rooms` — Список комнат
- `POST /api/chat/rooms` — Создать комнату

### Task Service
- `GET /api/tasks` — Список задач
- `POST /api/tasks` — Создать задачу
- `PUT /api/tasks/:id/complete` — Выполнить задачу
- `PUT /api/tasks/:id/confirm` — Подтвердить выполнение

## Технологии

- **Backend:** Go 1.26
- **Клиенты:** 
  - TUI: bubbletea, bubbles, lipgloss
  - Web: HTML5, CSS3, JavaScript (Vanilla)
- **База данных:** PostgreSQL 15
- **Кэш:** Redis 7
- **WebSocket:** gorilla/websocket
- **Контейнеризация:** Docker, Docker Compose

## Цветовая схема

Клиенты используют единую цветовую схему в стиле "матрицы":

| Элемент                | Цвет              | Hex              |
|------------------------|-------------------|------------------|
| Фон                    | Чёрный            | `#000000`        |
| Основной текст         | Зелёный           | `#00ff00`        |
| Заголовки              | Зелёный на чёрном | `#0f0` на `#000` |
| Сообщения пользователя | Циан              | `#0ff`           |
| Системные сообщения    | Зелёный           | `#0f0`           |
| Задачи                 | Жёлтый            | `#ff0`           |
| Ошибки                 | Красный           | `#f00`           |
