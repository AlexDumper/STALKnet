# STALKnet

Децентрализованный чат-сервис с системой задач и **real-time** обменом сообщениями. Поддерживает **консольный TUI-клиент** и **веб-клиент**.

## Клиенты

### Авторизация

STALKnet использует **централизованную систему авторизации** через Auth Service с JWT токенами.

**Требования к учетным данным:**
- **Имя пользователя:** минимум 2 символа
- **Пароль:** минимум 6 символов

**Процесс авторизации:**

1. Пользователь в режиме **Guest** имеет доступ только к `/help`
2. Отправка сообщений заблокирована до авторизации
3. После авторизации автоматически подключается **WebSocket** к чату

**Команды авторизации:**

| Команда | Описание |
|---------|----------|
| `/auth` | Начать процесс авторизации/регистрации |
| `/y` | Подтвердить создание профиля |
| `/n` | Отменить создание профиля |
| `/cancel` | Отменить текущий процесс авторизации |
| `/login <user> <pass>` | Быстрый вход для существующего пользователя |
| `/logout` | Выйти из системы (завершить сессию) |
| `/quit` | Выйти из аккаунта и приложения |

**Поток регистрации нового пользователя:**

```
1. /auth
   → "Введите имя сталкера:"

2. Введите имя (например: BG)
   → Проверка имени через API
   → Если имя занято → запрос пароля
   → Если имя свободно → "Создать профиль? /y или /n"

3. /y (для нового пользователя)
   → "Введите пароль для: BG"

4. Введите пароль (мин. 6 символов)
   → POST /api/auth/register → создание в PostgreSQL
   → POST /api/auth/login → получение JWT токенов
   → Создание сессии в Redis
   → Подключение WebSocket к чату
   → "Добро пожаловать, BG! Session ID: abc123..."
```

**Поток входа существующего пользователя:**

```
1. /auth
   → "Введите имя сталкера:"

2. Введите имя (например: BG)
   → Проверка имени через API
   → Имя найдено → "Введите пароль:"

3. Введите пароль
   → POST /api/auth/login → проверка пароля
   → Генерация JWT access/refresh токенов
   → Создание сессии в Redis
   → Подключение WebSocket к чату
   → "Вход выполнен успешно!"
```

**После авторизации:**
- Session ID отображается в шапке интерфейса
- WebSocket подключается к `ws://localhost:8083/ws/chat`
- Доступна отправка сообщений в реальном времени
- Доступны все команды

### Чат (Chat Service)

**Chat Service** (порт 8083) обеспечивает **real-time** обмен сообщениями между пользователями.

**Архитектура чата:**

```
┌─────────────┐     WebSocket      ┌──────────────┐
│ Web Client  │ ◄────────────────► │  Chat Hub    │
│  Browser    │  ws://:8083/ws     │  (in-memory) │
└─────────────┘                    └──────────────┘
                                          │
                                   ┌──────┴──────┐
                                   ▼             ▼
                            ┌──────────┐  ┌──────────┐
                            │  Room 1  │  │  Room 2  │
                            │ general  │  │  tasks   │
                            └──────────┘  └──────────┘
```

**Как работает:**

1. После авторизации клиент подключается к WebSocket endpoint
2. Hub регистрирует клиента в комнате (по умолчанию room_id=1)
3. Сообщения отправляются через WebSocket в формате JSON
4. Hub рассылает сообщение всем клиентам в комнате
5. При отключении клиент удаляется из комнаты

**Формат сообщений:**

```json
{
  "type": "message",
  "content": "Привет, сталкер!"
}
```

**Типы сообщений:**
- `message` — обычное сообщение пользователя
- `system` — системное сообщение (вход/выход)
- `task` — уведомление о задаче

**WebSocket подключение:**

```
GET /ws/chat?room_id=1&user_id=5&username=BG
```

**Параметры:**
- `room_id` — ID комнаты для подключения
- `user_id` — ID пользователя (из токена)
- `username` — Имя пользователя для отображения

## Быстрый старт

### Вариант 1: Запуск через Docker (рекомендуется)

```bash
# Запуск всех сервисов (PostgreSQL, Redis, Gateway, Auth, и др.)
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановка
docker-compose down
```

После запуска:
- **Web-клиент**: http://localhost:8080
- **Auth Service**: http://localhost:8081
- **PostgreSQL**: localhost:5432
- **Redis**: localhost:6379

### Вариант 2: Локальный запуск (для разработки)

Требуется установленные PostgreSQL 15 и Redis 7.

```bash
# 1. Инициализация БД (выполнить один раз)
psql -U postgres -f deploy/postgres/init.sql

# 2. Запуск Auth Service
cd services/auth
go run main.go

# 3. Запуск TUI клиента (в новом терминале)
cd client
go run main.go

# 4. Запуск Web-клиента (в новом терминале)
cd client
go run main_web.go
```

## API Endpoints

### Auth Service (порт 8081)

**Аутентификация и управление сессиями**

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/api/auth/register` | Регистрация нового пользователя |
| `POST` | `/api/auth/login` | Вход (получение JWT токенов) |
| `POST` | `/api/auth/logout` | Выход (инвалидация сессии) |
| `POST` | `/api/auth/refresh` | Обновление access токена |
| `POST` | `/api/auth/validate` | Проверка валидности токена |
| `POST` | `/api/auth/check-username` | Проверка существования пользователя |
| `GET` | `/api/auth/session` | Информация о текущей сессии |
| `GET` | `/health` | Проверка здоровья сервиса |

**Пример запроса регистрации:**
```bash
curl -X POST http://localhost:8081/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"BG","password":"password123"}'
```

**Ответ:**
```json
{
  "message": "User registered successfully",
  "user_id": 5,
  "username": "BG"
}
```

**Пример запроса входа:**
```bash
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"BG","password":"password123"}'
```

**Ответ:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
  "expires_in": 900,
  "user_id": 5,
  "username": "BG",
  "session_id": "abc123def456..."
}
```

**Пример проверки имени:**
```bash
curl -X POST http://localhost:8081/api/auth/check-username \
  -H "Content-Type: application/json" \
  -d '{"username":"BG"}'
```

**Ответ:**
```json
{
  "exists": true
}
```

### Static Content API (порт 8081)

**Получение статического контента из базы данных**

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/api/content/:key` | Получить контент по ключу |
| Query Param | `auth_state` | Статус авторизации (0=Guest, 4=Authorized) |

**Пример запроса:**
```bash
# Справка для гостя
curl "http://localhost:8081/api/content/help_guest?auth_state=0"

# Справка для авторизованного пользователя
curl "http://localhost:8081/api/content/help_authorized?auth_state=4"
```

**Ответ:**
```json
{
  "key": "help_guest",
  "type": "text",
  "title": "Базовые команды",
  "content": "╭────────────────────────────────────────────╮\n│ Доступные команды:\n│ /help - Показать эту справку\n│ ..."
}
```

**Таблица базы данных `static_content`:**
```sql
CREATE TABLE static_content (
    id SERIAL PRIMARY KEY,
    content_key VARCHAR(100) NOT NULL,      -- Уникальный ключ (help_guest, help_authorized)
    title VARCHAR(255),                      -- Заголовок
    content TEXT NOT NULL,                   -- Контент (текст справки)
    content_type VARCHAR(20) DEFAULT 'text', -- Тип: text, markdown, html
    min_auth_state INT DEFAULT 0,            -- Мин. статус авторизации
    max_auth_state INT DEFAULT 4,            -- Макс. статус авторизации
    language VARCHAR(10) DEFAULT 'ru',       -- Язык
    is_active BOOLEAN DEFAULT TRUE,          -- Активность
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

**Уровни доступа:**
- `0` — Guest (гость, не авторизован)
- `4` — Authorized (авторизованный пользователь)


### Chat Service (порт 8083)

**Real-time обмен сообщениями через WebSocket**

| Endpoint | Описание |
|----------|----------|
| `GET /ws/chat` | WebSocket подключение к чату |
| `GET /api/chat/rooms` | Список комнат |
| `POST /api/chat/rooms` | Создать комнату |
| `GET /api/chat/rooms/:id/messages` | Получить сообщения комнаты |
| `POST /api/chat/rooms/:id/messages` | Отправить сообщение |
| `GET /api/chat/rooms/:id/members` | Участники комнаты |
| `GET /health` | Проверка здоровья сервиса |

**WebSocket подключение:**

```javascript
const ws = new WebSocket(
  `ws://localhost:8083/ws/chat?room_id=1&user_id=5&username=BG`
);

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  console.log(`${msg.username}: ${msg.content}`);
};

ws.send(JSON.stringify({
  type: "message",
  content: "Привет, сталкер!"
}));
```

**Формат сообщения:**
```json
{
  "room_id": 1,
  "user_id": 5,
  "username": "BG",
  "content": "Привет, сталкер!",
  "type": "message",
  "timestamp": "2026-02-28T12:00:00Z"
}
```

**Типы сообщений:**
- `message` — сообщение пользователя
- `system` — системное уведомление
- `task` — уведомление о задаче

## Цветовая схема

Клиенты используют единую цветовую схему в стиле "матрицы":

| Элемент                | Цвет              | Hex              |
|------------------------|-------------------|------------------|
| Фон                    | Чёрный            | `#000000`        |
| Основной текст         | Зелёный           | `#00ff00`        |
| Заголовки              | Зелёный на чёрном | `#0f0` на `#000` |
| Сообщения пользователя | Циан              | `#0ff`           |
| Системные сообщения    | Зелёный           | `#0f0`           |
| Задачи                 | Жёлтый            | `#ff0`           |
| Ошибки                 | Красный           | `#f00`           |
